QA Java Project

Реализован сервис авторизации и аутентификации пользователей на базе JWT-токенов с контролем доступа по ролям (RBAC). Поддерживается регистрация, логин/логаут, управление пользователями и сессиями.
​
Sprint_6 — это учебный проект по UI-тестированию веб-сайта "Яндекс.Самокат" (почасовая/посуточная аренда с доставкой до метро). Написаны автоматические тесты на проверку страниц: домашняя, заказ самоката, личный кабинет. Создан чек-лист функционала перед релизом.
​
Инструменты:

Pytest (фреймворк тестов)

Selenium (автоматизация браузера)

Allure (красивые отчёты о тестах)

Page Object Model (POM) (структура кода: страницы как классы с методами)

Git/GitHub (версионность)
Установка: pip install -r requirements.txt, запуск: pytest tests --alluredir=allure_results, отчёт: allure serve allure_results.
​

Как это работает?
Тесты имитируют действия пользователя в браузере: открывают сайт, кликают кнопки, заполняют формы заказа самоката, проверяют результаты (время аренды, статус). Selenium управляет Chrome/Firefox, Pytest запускает сценарии последовательно, Allure собирает скриншоты/логи в HTML-отчёт.
​

Почему решение принято именно так?
POM выбран для читаемости: вместо хаоса селекторов в тестах — отдельные файлы на страницу (pages/utils). Selenium — стандарт для UI-тестов (работает с реальным браузером). Pytest + Allure — быстрая настройка, удобные отчёты для команды. Всё просто для спринта, без переусложнения.
​

Какие ошибки уже были совершены и чего стоит избегать?

Хардкод селекторов в тестах (решение: utils с константами).

Игнор нестабильности (длинные ожидания — использовать explicit waits в Selenium).

Нет smoke-тестов (избегать: группировать тесты по приоритету).

Отчёты не читаемы (Allure решает). Избегать: тестировать без фиксаторов багов (YouTrack), не коммитить secrets, всегда coverage >80%.​
